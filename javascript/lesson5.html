Урок 5: События и обработчики событий

Введение

События в JavaScript — это действия пользователя или браузера, такие как клики, ввод текста или наведение мыши. Обработчики событий позволяют вашему коду реагировать на эти действия, делая веб-страницы интерактивными. В этом уроке вы научитесь добавлять обработчики событий, работать с объектом события и создавать динамические интерфейсы.

Что такое события?

События — это сигналы, которые генерируются, когда пользователь взаимодействует со страницей (например, нажимает кнопку) или происходят изменения в браузере (например, страница загрузилась). JavaScript может "слушать" эти события и выполнять код в ответ.

Пример простого события:





const button = document.getElementById("myButton");
button.addEventListener("click", () => {
    alert("Кнопка нажата!");
});

В этом примере при клике на кнопку отображается всплывающее окно.

Добавление обработчиков событий

Есть два основных способа добавления обработчиков событий:

  • addEventListener: Современный и гибкий метод.
  • Атрибуты on...: Например, onclick в HTML (менее рекомендуемый).

Пример с addEventListener:

const button = document.getElementById("myButton");
button.addEventListener("click", () => {
    console.log("Клик!");
});

Пример с атрибутом onclick:





function handleClick() {
    console.log("Клик через onclick!");
}

addEventListener предпочтительнее, так как позволяет добавлять несколько обработчиков для одного события.

Типы событий

JavaScript поддерживает множество типов событий. Вот несколько популярных:

  • click: Клик мышью.
  • input: Изменение значения в поле ввода.
  • mouseover: Наведение мыши на элемент.
  • mouseout: Уход курсора с элемента.
  • submit: Отправка формы.
  • keydown: Нажатие клавиши.

Пример с mouseover и mouseout:

const div = document.getElementById("myDiv");
div.addEventListener("mouseover", () => {
    div.style.backgroundColor = "lightblue";
});
div.addEventListener("mouseout", () => {
    div.style.backgroundColor = "";
});

Этот код меняет цвет фона элемента при наведении мыши и возвращает исходный цвет при уходе курсора.

Объект события

Каждый обработчик события получает объект event, содержащий информацию о событии, например, какой элемент вызвал событие или какая клавиша была нажата.

const input = document.getElementById("myInput");
input.addEventListener("input", (event) => {
    console.log(`Введено: ${event.target.value}`);
});

Свойство event.target указывает на элемент, вызвавший событие, а event.target.value возвращает текущее значение поля ввода.

Пример с keydown:

document.addEventListener("keydown", (event) => {
    console.log(`Нажата клавиша: ${event.key}`);
});

Этот код выводит в консоль нажатую клавишу.

Предотвращение стандартного поведения

Некоторые события, такие как отправка формы, имеют стандартное поведение (например, перезагрузка страницы). Метод preventDefault позволяет отменить это поведение.




    
    




const form = document.getElementById("myForm");
form.addEventListener("submit", (event) => {
    event.preventDefault();
    console.log("Форма отправлена, но страница не перезагружена!");
});

Этот код предотвращает перезагрузку страницы при отправке формы.

Практика

1. Создайте кнопку, которая при клике меняет текст другого элемента на странице.

2. Добавьте обработчик события input для поля ввода, чтобы отображать введённый текст в реальном времени в другом элементе.

3. Создайте элемент, который меняет цвет фона при наведении мыши и возвращает исходный цвет при уходе курсора.

4. Создайте форму, которая при отправке выводит введённые данные в консоль и предотвращает перезагрузку страницы.

5. Напишите обработчик события keydown, который выводит сообщение, если пользователь нажимает клавишу Enter.
