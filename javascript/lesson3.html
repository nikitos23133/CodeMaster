Урок 3: Функции и области видимости

Введение

Функции в JavaScript позволяют группировать код для повторного использования, упрощая разработку. Область видимости определяет, где переменные доступны, а контекст выполнения влияет на значение this. В этом уроке вы научитесь создавать функции, работать с областью видимости и понимать контекст.

Создание функций

Функции объявляются с помощью ключевого слова function или как стрелочные функции. Они могут принимать параметры и возвращать значения с помощью return.

function greet(name) {
    return `Привет, ${name}!`;
}
console.log(greet("Алиса"));  // Вывод: Привет, Алиса!

Стрелочная функция — более краткий синтаксис:

const add = (a, b) => a + b;
console.log(add(5, 3));  // Вывод: 8

Параметры и значения по умолчанию

Функции могут иметь параметры с значениями по умолчанию:

function introduce(name, age = 18) {
    console.log(`Меня зовут ${name}, мне ${age} лет.`);
}
introduce("Боб");       // Вывод: Меня зовут Боб, мне 18 лет.
introduce("Алиса", 25); // Вывод: Меня зовут Алиса, мне 25 лет.

Переменное число аргументов

Используйте объект arguments или rest-параметры (...args) для работы с переменным числом аргументов:

function sum(...numbers) {
    return numbers.reduce((total, num) => total + num, 0);
}
console.log(sum(1, 2, 3));     // Вывод: 6
console.log(sum(10, 20, 30, 40)); // Вывод: 100

Область видимости

Область видимости определяет, где переменные доступны. В JavaScript есть три типа области видимости:

  • Глобальная: Переменные, объявленные вне функций, доступны везде.
  • Локальная (функциональная): Переменные внутри функции доступны только в ней.
  • Блочная: Переменные, объявленные с let или const внутри блока {}, доступны только в этом блоке.
let globalVar = "Я глобальная";

function testScope() {
    let localVar = "Я локальная";
    if (true) {
        let blockVar = "Я блочная";
        console.log(globalVar); // Вывод: Я глобальная
        console.log(localVar);  // Вывод: Я локальная
        console.log(blockVar);  // Вывод: Я блочная
    }
    // console.log(blockVar); // Ошибка: blockVar не определена
}
testScope();

Контекст выполнения и this

Ключевое слово this ссылается на объект, в контексте которого выполняется функция. Значение this зависит от того, как функция вызвана:

const person = {
    name: "Алиса",
    greet: function() {
        console.log(`Привет, я ${this.name}!`);
    }
};
person.greet();  // Вывод: Привет, я Алиса!

В стрелочных функциях this наследуется из окружающей области:

const person2 = {
    name: "Боб",
    greet: () => {
        console.log(`Привет, я ${this.name}!`);
    }
};
person2.greet();  // Вывод: Привет, я undefined! (this не привязан к person2)

Практика

1. Напишите функцию, которая принимает два числа и возвращает их произведение.

2. Создайте стрелочную функцию, которая принимает массив чисел и возвращает их среднее значение.

3. Напишите функцию, которая использует rest-параметры для объединения строк в одну с разделителем.

4. Создайте объект с методом, который использует this для вывода свойства объекта.

5. Продемонстрируйте разницу между глобальной, локальной и блочной областью видимости в одном примере.
