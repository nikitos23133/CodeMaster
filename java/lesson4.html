Урок 4: Наследование и полиморфизм

Введение

Наследование и полиморфизм — ключевые концепции объектно-ориентированного программирования в Java. Наследование позволяет создавать новые классы на основе существующих, а полиморфизм обеспечивает гибкость в вызове методов. В этом уроке вы изучите, как использовать наследование, переопределять методы и применять полиморфизм.

Наследование

Наследование позволяет классу (подклассу) унаследовать поля и методы другого класса (суперкласса) с помощью ключевого слова extends.

public class Animal {
    String name;

    public Animal(String name) {
        this.name = name;
    }

    void makeSound() {
        System.out.println("Какой-то звук");
    }
}

public class Dog extends Animal {
    public Dog(String name) {
        super(name);
    }

    @Override
    void makeSound() {
        System.out.println(name + " говорит: Гав!");
    }
}

public class Main {
    public static void main(String[] args) {
        Dog dog = new Dog("Бобик");
        dog.makeSound(); // Вывод: Бобик говорит: Гав!
    }
}

В этом примере класс Dog наследует от Animal и переопределяет метод makeSound. Ключевое слово super вызывает конструктор суперкласса.

Переопределение методов

Аннотация @Override указывает, что метод переопределяет метод суперкласса. Это улучшает читаемость и предотвращает ошибки.

public class Cat extends Animal {
    public Cat(String name) {
        super(name);
    }

    @Override
    void makeSound() {
        System.out.println(name + " говорит: Мяу!");
    }
}

Полиморфизм

Полиморфизм позволяет использовать объекты подклассов там, где ожидается суперкласс. Это достигается через динамическое связывание:

public class Main {
    public static void main(String[] args) {
        Animal animal1 = new Dog("Бобик");
        Animal animal2 = new Cat("Мурка");
        animal1.makeSound(); // Вывод: Бобик говорит: Гав!
        animal2.makeSound(); // Вывод: Мурка говорит: Мяу!
    }
}

Переменные типа Animal могут ссылаться на объекты Dog или Cat, а вызов метода определяется типом объекта.

Абстрактные классы

Абстрактные классы не могут быть инстанцированы и используются как шаблоны для подклассов:

public abstract class Animal {
    String name;

    public Animal(String name) {
        this.name = name;
    }

    abstract void makeSound();
}

public class Dog extends Animal {
    public Dog(String name) {
        super(name);
    }

    @Override
    void makeSound() {
        System.out.println(name + " говорит: Гав!");
    }
}

Метод makeSound объявлен как abstract, и подкласс обязан его реализовать.

Интерфейсы

Интерфейсы определяют контракт, который классы должны реализовать с помощью ключевого слова implements:

public interface Speakable {
    void speak();
}

public class Dog extends Animal implements Speakable {
    public Dog(String name) {
        super(name);
    }

    @Override
    void makeSound() {
        System.out.println(name + " говорит: Гав!");
    }

    @Override
    public void speak() {
        makeSound();
    }
}

Практика

1. Создайте класс Vehicle и подкласс Car, переопределив метод для вывода типа транспортного средства.

2. Создайте абстрактный класс Shape с абстрактным методом getArea. Реализуйте его в подклассах Circle и Rectangle.

3. Напишите программу, демонстрирующую полиморфизм с массивом объектов разных подклассов.

4. Создайте интерфейс Printable с методом print и реализуйте его в классе Book.
