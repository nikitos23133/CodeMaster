Урок 5: Исключения и обработка ошибок

Введение

Исключения в Java — это события, которые нарушают нормальное выполнение программы, например, деление на ноль или доступ к несуществующему файлу. Обработка исключений позволяет управлять такими ошибками, делая программы надёжными. В этом уроке вы научитесь использовать блоки try-catch, обрабатывать разные типы исключений и создавать пользовательские исключения.

Что такое исключения?

Исключения — это объекты, представляющие ошибки. Они делятся на два типа:

  • Checked: Проверяемые исключения, которые компилятор требует обработать (например, IOException).
  • Unchecked: Непроверяемые исключения, такие как ArithmeticException, которые возникают во время выполнения.
public class Main {
    public static void main(String[] args) {
        int a = 10;
        int b = 0;
        // System.out.println(a / b); // Ошибка: ArithmeticException
    }
}

Блоки try-catch

Блок try-catch используется для обработки исключений:

public class Main {
    public static void main(String[] args) {
        try {
            int a = 10;
            int b = 0;
            System.out.println(a / b);
        } catch (ArithmeticException e) {
            System.out.println("Ошибка: Деление на ноль!");
        }
    }
}

Код в блоке try выполняется, а если возникает исключение, управление передаётся блоку catch.

Множественные catch-блоки

Можно обрабатывать несколько типов исключений:

public class Main {
    public static void main(String[] args) {
        try {
            int[] array = {1, 2, 3};
            System.out.println(array[5]); // ArrayIndexOutOfBoundsException
            int result = 10 / 0; // ArithmeticException
        } catch (ArithmeticException e) {
            System.out.println("Деление на ноль!");
        } catch (ArrayIndexOutOfBoundsException e) {
            System.out.println("Выход за пределы массива!");
        }
    }
}

Блок finally

Блок finally выполняется всегда, независимо от того, было ли исключение:

public class Main {
    public static void main(String[] args) {
        try {
            int a = 10;
            int b = 0;
            System.out.println(a / b);
        } catch (ArithmeticException e) {
            System.out.println("Ошибка: " + e.getMessage());
        } finally {
            System.out.println("Операция завершена.");
        }
    }
}

Создание пользовательских исключений

Вы можете создать собственные исключения, унаследовав от класса Exception:

public class InvalidAgeException extends Exception {
    public InvalidAgeException(String message) {
        super(message);
    }
}

public class Main {
    public static void checkAge(int age) throws InvalidAgeException {
        if (age < 18) {
            throw new InvalidAgeException("Возраст должен быть 18 или больше!");
        }
        System.out.println("Доступ разрешён.");
    }

    public static void main(String[] args) {
        try {
            checkAge(15);
        } catch (InvalidAgeException e) {
            System.out.println("Ошибка: " + e.getMessage());
        }
    }
}

Ключевое слово throws указывает, что метод может выбросить исключение.

Практика

1. Напишите программу, которая делит два числа и обрабатывает исключение деления на ноль.

2. Создайте программу, которая получает элемент массива по индексу и обрабатывает выход за пределы массива.

3. Напишите метод, который проверяет строку на null и выбрасывает NullPointerException с пользовательским сообщением.

4. Создайте пользовательское исключение InvalidInputException и используйте его для проверки ввода числа в диапазоне.
